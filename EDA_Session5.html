  • Motivation:

In the last assigment we worked with a Health Care dataset that inlcuded values of several providers on different states in the US. The dataset also included average charges, total payments and medicare payments per porivder id. def stat_flat(data,group_var1,cont_var,group_var2=False):
    if group_var2 == False:
        coln =list(data[[group_var1,cont_var]].describe().index)
        row = data[[group_var1]].drop_duplicates().reset_index(drop=True).values.tolist()
        row = [item for sublist in row for item in sublist] #faltten the list
        desc_cont_var = pd.DataFrame(data[[group_var1,cont_var]].groupby(group_var1).describe().values.tolist(), columns=coln,index=row)
    elif group_var2 != False:
        coln =list(data[[group_var1,group_var2,cont_var]].describe().index)
        row = data[[group_var1,group_var2]].drop_duplicates().reset_index(drop=True).values.tolist()
        row_s = [sublist[0] for sublist in row] #faltten the list
        row2= pd.DataFrame([sublist[1] for sublist in row])
        desc_cont_var = pd.DataFrame(data[[group_var1,group_var2,cont_var]].groupby([group_var1,group_var2]).describe().values.tolist(), columns=coln,index=row_s,)
        row2 =row2.set_index(desc_cont_var.index)
        desc_cont_var.insert(loc=0,column=str(group_var2),value=row2)
    return desc_cont_var

K-Means Clustering¶

The method picked for clustering these observations is K-Means clustering. This method works to sepprate observations into specific clusters based on the characteristics of their variables. In each iteration of the algorithm observations are relocated to different clusters and a cluster centroid is created. The goal of the algorithm is to create centroids that have a total sum of means within each cluster as small as possible and a total sum of means from cluster to cluster as large as possible.

In order to decide the ideal number of clusters for this dataset lets run a total within square sum for a k-means of 1 cluster all the way to a k-means of 11 clusters. The idea is to graph the within cluster sum of squares agains the number of clusters of the k-means algo. The obejctive is to idnetify a number of cluster where the most notorious change from one cluster to another occurs. This big change is called the elbow method.

LDA¶

In order to visualize the data in a 2D plot we need to reduce the dimensions of the datset from 5 variables to 2 variables. There are several methods used for dimension reduction and all have their pros and cons. One method I like using for clustering is LDA. The goal of LDA is to " Maximize the function that represents the difference between the means, normalized by a measure of the within-class variability" where class variability in this case is a target variable. In this case the target variable is the cluster number assigned to the observation. However, LDA has a caveat, it only performs well with normally distributed data. But since we are both normalizing and scaling the data before inputing it into the LDA model then LDA seems like the right method for this job.

Using Z-Score to Scale the Data¶

It seems that the data is still very much spread out across the board. Lets look at normalizing and scaling this dataset. Becuase we are already dealing with ratios, the data is normalized. So lets move into scaling it using a z-score.

A z-score calculates the number of standard deviations that a single point is from the mean. This helps scale data while also mantaining its underlying behavior.
